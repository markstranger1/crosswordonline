# Автоматизированная система составления и решения классического кроссворда

## 📌 Описание проекта
Автоматизированная информационная система, предназначенная для формирования и решения классических кроссвордов.  
Система поддерживает режимы генерации кроссвордов, их решения пользователем, а также хранение и обработку данных предметной области.

Проект выполнен в рамках учебной дисциплины и ориентирован на формализацию требований, проектирование структуры системы и моделирование её поведения.

---

## 🎯 Цели проекта
- автоматизация процесса составления кроссвордов;
- формализация правил генерации и решения кроссвордов;
- проектирование информационной системы с чёткой моделью данных;
- разработка UML-моделей для описания структуры и поведения системы.

---

## 👥 Роли пользователей
В системе предусмотрена ролевая модель доступа:

- **Пользователь**
  - решение кроссвордов;
  - взаимодействие с элементами кроссворда;
  - получение подсказок;
  - сохранение прогресса.

- **Администратор**
  - управление словарями и вопросами;
  - контроль корректности данных;
  - настройка параметров генерации кроссвордов.

---

## ⚙️ Основной функционал
- автоматическое формирование кроссвордов на основе словаря;
- ручной режим заполнения кроссворда;
- решение кроссвордов пользователем;
- проверка корректности введённых данных;
- сохранение состояния и результатов;
- разграничение прав доступа.

---

## 🏗 Архитектура системы
Система спроектирована с использованием клиент-серверной архитектуры.

Основные компоненты:
- клиентская часть (пользовательский интерфейс);
- серверная часть (обработка логики и правил);
- база данных для хранения элементов кроссворда и пользовательских данных.

---

## 📐 UML-моделирование
В ходе проектирования системы были разработаны UML-диаграммы, отражающие различные аспекты её функционирования:

- диаграмма вариантов использования (Use Case);
- диаграмма классов;
- диаграмма последовательности;
- диаграмма компонентов;
- диаграмма развёртывания.

---

## 🗄 Модель данных
Спроектирована логическая и физическая модель базы данных, обеспечивающая:
- хранение слов, вопросов и ответов;
- представление структуры кроссворда;
- сохранение пользовательских решений и прогресса.
